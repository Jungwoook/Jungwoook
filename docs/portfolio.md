# 포트폴리오

데이터 처리, 로그 분석, 실시간 상태 전달, 파일 저장 구조 설계를 중심으로 백엔드 시스템을 개발해 왔습니다. 아래 프로젝트들은 관련 경험을 보여줄 수 있는 대표 사례입니다.

## 주요 실무 프로젝트

### 자동화 테스트 웹 모니터링 서비스

자동화 테스트 실행 결과와 로그 데이터를 수집, 분석하고 대시보드로 시각화한 모니터링 시스템입니다. 로그 데이터를 단순 조회가 아니라 통계와 분석에 활용할 수 있는 형태로 가공하는 구조를 구현했습니다.

- **기간**: 2023.01 ~ 2024.02
- **역할**: 백엔드 개발
- **핵심 키워드**: 로그 데이터 처리, 통계 집계, 대시보드 API, 응답 최적화, 반복 오류 분석
- **상세 문서**: [자동화 테스트 웹 모니터링 서비스](./projects/automated-test-monitoring.md)

### 자동화 테스트 클라이언트

자동화 테스트 실행과 로그 수집을 지원하는 Python 기반 클라이언트입니다. 테스트 실행 UI, 시나리오 조회, 실행 로그 저장 기능을 통해 테스트 수행 흐름을 지원했습니다.

- **기간**: 2022.09 ~ 2023.01
- **역할**: 클라이언트 개발
- **핵심 키워드**: PyQt, 테스트 실행 UI, 로그 수집, 시나리오 조회, DB 연동

## 교육/팀 프로젝트

### AI 기반 프레임워크 변환 플랫폼

eGovFrame과 Spring Boot 간 코드 변환을 자동화하고, 변환 진행 상태와 결과 파일을 웹에서 확인할 수 있는 플랫폼입니다. 실시간 상태 조회와 파일 저장 구조를 중심으로 백엔드 기능을 구현했습니다.

- **기간**: 2025.06 ~ 2025.09
- **역할**: 백엔드 개발
- **핵심 키워드**: Spring Boot, SSE, AWS S3, Presigned URL, MySQL/MongoDB 분리
- **상세 문서**: [AI 기반 프레임워크 변환 플랫폼](./projects/ai-framework-conversion-platform.md)

## 개인 프로젝트

### 로그 분석 프로젝트

업로드된 로그 파일을 파싱하고 API Key, 서비스 ID, 브라우저 사용 비율 등 주요 통계를 JSON으로 반환하는 Spring Boot 기반 로그 분석 API입니다. Gradle 멀티모듈 구조로 HTTP 계층, 파싱 계층, 분석 계층을 분리하고, 전체 로그를 메모리에 적재하지 않는 스트리밍 기반 누적 분석 흐름을 구성했습니다.

- **기간**: 개인 프로젝트
- **역할**: 개인 개발
- **핵심 키워드**: Spring Boot, Java 17, Gradle Multi Module, 로그 파싱, 스트리밍 분석, 통계 처리
- **상세 문서**: [Log Analyzer](./projects/log-analyzer.md)

### GitHub 이슈관리 프로젝트

GitHub 이슈와 작업 흐름을 관리하는 개인 프로젝트입니다. 이슈 추적, 작업 상태 관리, 협업 흐름 개선을 중심으로 정리할 예정입니다.

- **기간**: 작성 예정
- **역할**: 개인 개발
- **핵심 키워드**: GitHub API, 이슈 관리, 작업 흐름, 협업 관리
- **상세 문서**: 작성 예정
